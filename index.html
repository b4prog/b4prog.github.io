Analyze GraphQL requests in HAR files without leaving the browser

GraphQL HAR Analyzer is a browser-based tool for frontend engineers, API developers, QA analysts, and support teams who need to inspect captured GraphQL traffic quickly.

Use it to review production captures, validate schema coverage, investigate failing operations, and understand how GraphQL requests behave across real user sessions.

Everything runs locally in your browser session. HAR files, GraphQL schemas, request headers, and response payloads stay on the page and are never uploaded by the app.

Key capabilities

  • Load a HAR export and isolate GraphQL requests from the rest of the network log.
  • Map captured requests back to a GraphQL schema to understand operation coverage.
  • Filter, sort, and inspect requests by operation type, operation name, status, and error state.
  • Compare raw request details with schema context to debug API behavior faster.

How to use it

  1. Load a GraphQL schema file when you want operation names and fields correlated against the captured traffic.
  2. Load a HAR file exported from your browser or monitoring workflow.
  3. Review the request list, inspect matching operations, and use filters to narrow down failing or unexpected traffic.

Frequently asked questions

What is a HAR file?

A HAR file is an HTTP Archive export from a browser or proxy tool. It records network requests and responses, timing details, headers, and payload metadata so you can inspect what happened during a session.

What is GraphQL HAR Analyzer used for?

It helps engineers inspect HAR captures, extract GraphQL requests, correlate them with a schema, and troubleshoot API traffic from real browser sessions.

Can I analyze GraphQL HAR files without uploading them?

Yes. The application runs entirely in the browser, so HAR files and schema files stay local to your session.

How do I save a HAR file?

Open your browser's developer tools, go to the Network panel, reproduce the page activity you want to capture, then use the export or save option such as "Save all as HAR" or "Export HAR". Most browsers let you do this from the Network panel context menu or toolbar.

Who is this GraphQL debugging tool for?

It is useful for frontend engineers, platform teams, QA analysts, support engineers, and anyone investigating GraphQL requests captured in HAR exports.

Do I need a backend or account to use the analyzer?

No. The site is a static GitHub Pages deployment and the analysis happens in the browser without a login or backend service.
